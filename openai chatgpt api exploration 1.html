This is my first walkthrough and exploration of OpenAI's API via the OpenAI Python API. Some of this compares to my first HuggingFace walthrough/exploration, but this isn't an apples-to-apples comparison, as with HuggingFace I was just trying out several (much smaller) open source models locally and this walk through is relying on OpenAI's chatGPT which is a huge model that's been fine tuned in many ways for many different tasks and is running on OpenAI's infrastucture. I put this together to help friends and colleagues better understand what is possible in AI, ML, and particularly with transformer architectures and LLMs. Those with little to no programming experience can still probably follow the video, as I try to explain what's happening. For those who code, I suggest downloading the notebook and putting the video on in the background as you work through it yourself. You can also just talk directly to chatGPT.

This content is a mix of my own exploration plus several examples from or inspired by ChatGPT Prompt Engineering for Developers at deeplearning.ai by Isa Ford of OpenAI and Andrew Ng of deeplearning.ai.

1. Getting Started¶

To use the api you'll need to purchase some credits. I've been playing with chatGPT for hours and spent about 12 cents already, so you can start with the minimum purchase which I think is $5 one-time cost. Check out pricing to pay and get an OPENAI_API_KEY. Note that I mostly stick to the discount model, gpt-3.5-turbo, which is charged by token. If tokens are new to you, most tokens are small words or parts of words, such that you'll typically use 2-3 tokens for every word you use. By my math, 1 cent gets you 200,000 input tokens or 66,000 output tokens. So if you input a 10,000 word article and ask for a 1,000 word summary you might spend close to 1/5th of a cent.

You'll also need to install the openai python library with your favorite package manager or or use pip install openai

In conda, I did this:

conda create --name mynewenv
conda activate mynewenv
conda install openai

Once you get your API key add it to your environment with something like export OPENAI_API_KEY="blah". Yeah, you could hardcode it, but you're better than that.
